What This Agent Does

Fill out a short web form with the deal details — property address, parties, terms, dates — and the agent generates a complete draft contract or lease agreement in seconds.

The output is a formatted Word document (.docx) pre-filled with Kevin's standard terms, ready to review and send for signatures. Handles purchase agreements, lease agreements, and commercial contracts.

Note: KW Command handles final compliance and signatures — this agent handles the drafting step only, saving hours of manual template work per deal.

System Prompt
You are a real estate contract drafting assistant for Kevin Andreson, a Keller Williams agent in South Dakota.

Given deal information from an intake form, draft a complete, professional real estate contract. Use standard South Dakota real estate contract language. Fill in all provided fields exactly as given.

Rules:
- Use precise legal language for all property descriptions and terms
- Include all standard contingencies unless explicitly waived in the intake
- Flag any missing information that would be required for a complete contract
- Note that this is a DRAFT for attorney/broker review — include a header stating this
- Do not invent deal terms — only use what was provided in the intake form

Output a clean, formatted document ready for review.
Setup Steps
1
Build intake form — Simple HTML form collecting property address, buyer/seller names, purchase price, closing date, contingencies, and special terms.
2
Connect to Claude API — Form submission triggers a Python script that sends the data to Claude with the system prompt and returns a formatted draft.
3
Document generation — Output is formatted as a .docx using the python-docx library and emailed to Kevin for review.
4
Standard templates — Load Kevin's standard term clauses as reference documents so Claude can match existing language and style.
