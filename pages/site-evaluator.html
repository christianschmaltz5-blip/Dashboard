Setting Up

Development Site Evaluator

Monitors Rapid City and Pennington County planning channels for new development activity — and evaluates specific parcels on demand with a full feasibility report.

What This Agent Does

Two functions in one agent:

1. Weekly monitoring — Checks Pennington County and Rapid City planning sources on a set schedule (see below). When something new appears — a rezoning, a plat, a conditional use permit, a major permit — it sends an alert so Kevin can act before it hits the open market.

2. On-demand site analysis — Input any parcel address or APN in Rapid City, SD, and the agent returns a complete development feasibility report: zoning classification, comparable land sales, proforma model with estimated costs and returns, and a plain-English go/no-go recommendation.

On-Demand Report Sections
Site Overview
Address, APN, acreage, current zoning, allowable uses, utility access — pulled from Pennington County GIS.
Comparable Sales
3–5 recent land sales within 5 miles, adjusted for size and zoning. Establishes land value range.
Proforma Model
Estimated development cost, projected NOI, cap rate, and IRR for the most likely use case.
Go / No-Go
Plain-English recommendation with key risk factors and the single biggest reason to proceed or pass.

System Prompt
You are a real estate development analyst for Kevin Andreson, focused on Rapid City and Pennington County, South Dakota.

MONITORING MODE — run on the schedule above:
Check the assigned planning sources for new activity since the last check. For each new item:
1. Address or parcel identifier
2. Type of activity (rezoning, permit, plat, conditional use, etc.)
3. Current status
4. Why it matters (development opportunity, infrastructure signal, competitor activity)
5. Recommended action

SITE ANALYSIS MODE — triggered by a parcel address or APN:
Produce a development feasibility report:
1. Site Overview — zoning, acreage, current use, utility access (Pennington County GIS)
2. Comparable Sales — 3-5 recent sales of similar parcels, adjusted for size and location
3. Proforma Model — for the highest-and-best use:
   - Estimated land cost (from comps)
   - Estimated hard construction cost ($/sqft × buildable area)
   - Soft costs (15% of hard)
   - Total project cost
   - Projected rental income or sale price at completion
   - NOI, cap rate, projected IRR
4. Go / No-Go — one paragraph, plain English

Be conservative in estimates. Flag assumptions clearly. If data is unavailable, say so rather than guessing.
Setup Steps
1
Web scrapers — Build Python scrapers for pennco.org/planning and boxeldersd.gov that detect new agenda items, permits, and documents since the last run using content hashing.
2
Scheduler — Add Monday, Tuesday, Wednesday, Thursday jobs to the macOS launchd schedule alongside the market report.
3
GIS parcel lookup — Connect to Pennington County GIS for on-demand parcel data: zoning, ownership, acreage, utility access.
4
Alert email — Send formatted summaries to arecblackhills@gmail.com using the same Gmail setup as the market report — only when new items are detected.
