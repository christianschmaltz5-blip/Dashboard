# Kevin Andreson Operations Dashboard

Internal automation dashboard for Kevin Andreson, Keller Williams Realty Black Hills.
Static HTML site (served via GitHub Pages) + standalone Python agents that generate
its data and run on a schedule.

## Setup

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Requires **Python 3.14+** (scripts were built/run against
`/Library/Frameworks/Python.framework/Versions/3.14/bin/python3`).

### Secrets (not in git — must be provided separately)

Each agent folder that talks to an external service expects its own `config.py`
and/or `.env`, all gitignored. These are **not included in this repo** and must
be handed off through a secure channel (password manager, not email/Slack):

| File | Contains |
|---|---|
| `agents/mls_report/config.py` | IMAP app password for `arecblackhills@gmail.com`, price bands / property type config |
| `agents/mls_report/.env` | Paragon MLS website logins (Black Hills + Mount Rushmore boards) |
| `agents/*/config.py` (others) | Recipient lists, API keys, board-specific settings — check each folder |
| Anthropic API key | Needed by any agent that calls Claude (zoning summarization, newsletter drafting, dev-watch digests) |

Without these, the agent scripts will fail to authenticate — the static site
itself will still load fine, it just won't get new data.

## Running the site

It's static HTML — no server needed. Open `index.html` directly, or it's
deployed via GitHub Pages from this repo.

## Running an agent manually

```bash
cd agents/mls_report
python3 parse_paragon_inbox.py          # pulls new Paragon listing emails → js/paragon-listings.js
python3 parse_paragon_inbox.py --check  # dry run, no writes
python3 mls_report.py --send            # sends the market update email
python3 health_check.py                 # pipeline health check
```

Same pattern for `agents/zoning_monitor/monitor.py`, `agents/box_elder_prospects/monitor.py`,
`agents/buyer_match/buyer_match.py`, `agents/hwy1416_dev_watch/monitor.py`,
`agents/hwy1416_newsletter/send_newsletter.py`, `agents/listing_newsletter/generate_newsletter.py`.

to a new machine does **not** move the schedule — recreate equivalents there
(launchd on Mac, cron/systemd timers on Linux, or a hosted cron runner):
